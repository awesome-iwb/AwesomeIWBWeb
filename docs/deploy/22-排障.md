# 排障清单

## 1. 访问首页正常但 /api 报错

检查：
- Nginx 是否配置了 `/api` 反代
- 后端是否在运行（systemd/面板守护/容器）
- 后端监听端口是否为 8080

命令：

```bash
curl -sS http://127.0.0.1:8080/api/projects | head
sudo nginx -t
```

## 2. 详情页显示 404 / 项目列表缺失

常见原因：
- 你启用了 SSG 静态路由，但新增项目只写入了 `backend/runtime/data.json`，没有同步到 `backend/src/data.json`

说明：
- 前端 `vite-ssg` 构建会基于 seed 文件生成路由列表
- 如果生产环境是“纯静态托管”，新增项目后需要重新构建前端并发布 `frontend/dist`

## 3. 后端启动后立即退出

### systemd

```bash
sudo systemctl status awesome-iwb-backend --no-pager
sudo journalctl -u awesome-iwb-backend -n 200 --no-pager
```

### Docker

```bash
docker compose logs -n 200 backend
```

## 4. DB 模式连接失败

检查 `DATABASE_URL` 是否正确：
- 用户名/密码
- host/port
- 数据库名

快速验证：

```bash
psql "$DATABASE_URL" -c 'select 1;'
```

## 5. HTTPS 证书申请失败

检查：
- 域名解析是否已生效
- 80 端口是否放通（Let’s Encrypt HTTP-01）
- 站点是否能通过 HTTP 访问
