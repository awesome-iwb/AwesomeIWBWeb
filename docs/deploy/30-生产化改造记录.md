## install.sh 增量记录（本轮）

- 增加系统 PG14 释放逻辑：检测 `postgresql` 服务运行时执行 `stop + disable`（不卸载）。
- 执行顺序收敛为：root 校验 -> 用户与目录幂等创建 -> apt 依赖 -> 停系统 PG -> Docker -> Bun -> 拉代码 -> 起 PG 容器 -> migrate -> build -> systemd -> nginx。
- 增加 `--with-https` 可选段，支持 `SSL_PROVIDER=letsencrypt|cloudflare-origin`。
- 新增 Cloudflare Real IP 刷新脚本与可选防火墙收敛脚本。

## PG16 兼容性核查

- 核查结果：后端 SQL 与迁移未使用 PG16 专属语法（如 `MERGE`、`pg_stat_io`、`uuidv7()` 等）；当前 DDL/DML 在 PG14/PG16 兼容范围内。
- 结论：切换到 `postgres:16-alpine` 无代码级阻断风险。

## 数据迁移与种子

- 运行时迁移：`backend/src/scripts/migrate-runtime-to-pg.ts`（幂等导入 runtime JSON）
