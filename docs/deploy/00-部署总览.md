# 部署总览

> 上线必读：先阅读 `deploy/README.md` 与 `docs/deploy/30-生产化改造记录.md`，按生产鉴权与初始化要求执行。
> 生产环境已停用 JSON 模式（仅开发可用），`DATABASE_URL` 缺失将直接拒绝启动。
> INFO：用户已明确接受 root+密码登录风险，部署流程不修改 SSH 配置。

## 部署形态

当前唯一主线：**应用原生（systemd + nginx）+ DB Docker（PostgreSQL 16）**。
前端/后端/Nginx 应用层 Docker 章节均视为废弃方案，仅作历史参考，不作为上线步骤。

本文档面向要把 AwesomeIWBWeb 部署到服务器（含面板环境）的维护者。

> 已停用：前端/后端/nginx 容器化编排（仅保留数据库容器）。
> 已停用：生产 JSON 模式（仅开发可用），生产必须配置 PostgreSQL。

## 1. 系统结构

- 前端：`frontend/`（Vue 3 + Vite + vite-ssg）
  - 生产建议：构建后生成 `frontend/dist`，由 Nginx 静态托管
- 后端：`backend/`（Bun + Elysia）
  - 监听端口：默认 `8080`
  - 对外建议只通过 Nginx 暴露 `/api/*`

## 2. 后端运行模式（必须理解）

后端支持两种数据模式，由环境变量 `DATABASE_URL` 决定：

- DB 模式（生产唯一允许）
  - 设置 `DATABASE_URL` 后即启用 PostgreSQL
  - 启动时会自动执行 migrations（无需手工跑迁移命令）
- JSON 模式（仅开发）
  - 不设置 `DATABASE_URL`
  - 数据落盘：`backend/runtime/*.json`

生产部署强烈建议使用 DB 模式，以避免 runtime 文件在多进程、面板守护、备份恢复时出现一致性问题。

## 3. 端口与网络

最典型的生产拓扑（单机）：

- Nginx：80/443（对外）
- 后端：127.0.0.1:8080（仅本机）
- PostgreSQL：127.0.0.1:5432（仅本机，或容器网络）
