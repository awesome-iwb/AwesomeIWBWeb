# 宝塔面板（BT）部署教程

目标：
- 使用宝塔管理 Nginx、站点、HTTPS、守护进程
- 后端使用 DB 模式（PostgreSQL）
- 前端静态文件由宝塔 Nginx 直接托管

## 0. 前置条件

- 已安装宝塔面板（建议 Linux x86_64）
- 已准备域名并解析到服务器

## 1. 安装基础软件（宝塔软件商店）

在宝塔软件商店安装：
- Nginx
- PostgreSQL（如果宝塔没有 PostgreSQL 插件，可改用 Docker 方式，见下文）

## 2. 部署代码

建议目录：`/www/wwwroot/awesome-iwb`

宝塔终端执行：

```bash
cd /www/wwwroot
git clone <YOUR_REPO_GIT_URL> awesome-iwb
```

## 3. PostgreSQL（两种方式）

### A) 宝塔原生 PostgreSQL（若可用）

1) 在 PostgreSQL 中创建用户/库：

- 用户：`awesome_iwb`
- 数据库：`awesome_iwb`
- 密码：请使用强密码

2) 记下连接串（后面要用）：

```text
postgres://awesome_iwb:<PASSWORD>@127.0.0.1:5432/awesome_iwb
```

### B) Docker 启动 PostgreSQL（推荐兼容）

如果宝塔没有 PostgreSQL 插件或你不想用插件：

```bash
cd /www/wwwroot/awesome-iwb/backend
docker compose up -d
```

然后使用 compose 里的用户名密码拼接 `DATABASE_URL`。

## 4. 构建前端（生成 dist）

在宝塔终端：

```bash
cd /www/wwwroot/awesome-iwb/frontend
bun install --no-progress
bun run build
```

构建产物：`/www/wwwroot/awesome-iwb/frontend/dist`

## 5. 创建站点（宝塔 → 网站）

1) 网站 → 添加站点
- 域名：`example.com`
- 根目录：建议指向 `/www/wwwroot/awesome-iwb/frontend/dist`

2) 设置 SSL
- 网站 → SSL → Let’s Encrypt → 申请

## 6. 配置反向代理（把 /api 转到后端）

网站 → 设置 → 反向代理 → 添加反向代理：

- 代理名称：`awesome-iwb-api`
- 目标 URL：`http://127.0.0.1:8080`
- 发送域名：`$host`
- 代理目录：`/api`

## 7. 启动后端（Bun）并守护

### 1) 安装 Bun

在宝塔终端：

```bash
curl -fsSL https://bun.sh/install | bash
```

确保 `bun` 可用：

```bash
~/.bun/bin/bun -v
```

### 2) 安装后端依赖

```bash
cd /www/wwwroot/awesome-iwb/backend
bun install --no-progress
```

### 3) 配置环境变量

建议写入一个文件（宝塔上也方便排查）：

`/www/wwwroot/awesome-iwb/backend/.env`

```bash
DATABASE_URL=postgres://awesome_iwb:<PASSWORD>@127.0.0.1:5432/awesome_iwb
```

### 4) 用宝塔“守护进程”启动

宝塔 → 软件商店 → 搜索并安装（或打开）：
- PM2 管理器 或 守护进程（不同版本入口略有差异）

添加一个守护进程：
- 启动目录：`/www/wwwroot/awesome-iwb/backend`
- 启动命令：`/root/.bun/bin/bun run start`
- 环境变量：把 `DATABASE_URL` 填进去（或在命令前 `export`）

启动后检查：
- 访问 `http://127.0.0.1:8080/api/projects` 是否有返回
- 访问站点 `https://example.com/` 是否正常

## 8. 升级与回滚（简要）

升级：

```bash
cd /www/wwwroot/awesome-iwb
git pull
cd backend && bun install --no-progress
cd ../frontend && bun install --no-progress && bun run build
```

然后在宝塔里重启后端守护进程，并重载 Nginx。
