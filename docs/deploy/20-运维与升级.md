# 运维与升级

## 1. 常用检查

### 后端健康检查

```bash
curl -sS http://127.0.0.1:8080/api/projects | head
```

### Nginx 配置检查

```bash
sudo nginx -t
sudo systemctl reload nginx
```

### DB 模式确认

DB 模式：项目对象通常包含 `id`/`slug` 字段；JSON 模式通常没有。

```bash
curl -sS http://127.0.0.1:8080/api/projects | python3 - <<'PY'
import sys, json
j=json.load(sys.stdin)
first=None
for c in j.get('categories', []):
  ps=c.get('projects') or []
  if ps:
    first=ps[0]
    break
print('has_id', isinstance(first, dict) and 'id' in first)
print('has_slug', isinstance(first, dict) and 'slug' in first)
PY
```

## 2. 代码升级（原生部署）

```bash
cd /opt/awesome-iwb
git pull
```

### 后端

```bash
cd backend
bun install --no-progress
sudo systemctl restart awesome-iwb-backend
```

说明：
- DB 模式会在启动时自动跑 migrations

### 前端

```bash
cd ../frontend
bun install --no-progress
bun run build
sudo systemctl reload nginx
```

## 3. 代码升级（面板部署）

原则同上，只是“重启后端”的动作由面板守护/PM2 完成。

## 4. 日志查看

### systemd（原生部署）

```bash
sudo journalctl -u awesome-iwb-backend -n 200 --no-pager
```

### Docker（容器部署）

```bash
docker compose logs -n 200 backend
docker compose logs -n 200 postgres
```
