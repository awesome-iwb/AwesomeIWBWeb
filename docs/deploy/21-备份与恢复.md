# 备份与恢复

本文覆盖两类备份：
- 数据库（DB 模式必备）
- 站点资源（代码、配置、静态文件）

## 1. 数据库备份（PostgreSQL）

### 1) 逻辑备份（推荐）

```bash
pg_dump "$DATABASE_URL" -Fc -f awesome_iwb.dump
```

优点：
- 可跨版本迁移
- 可按需恢复

### 2) 恢复

```bash
pg_restore -d "$DATABASE_URL" --clean --if-exists awesome_iwb.dump
```

## 2. 静态文件备份

生产部署一般会有：
- `/opt/awesome-iwb`（代码）
- `/etc/nginx/sites-available/awesome-iwb.conf`（Nginx 配置）
- `/etc/awesome-iwb/backend.env`（后端环境变量）

建议做法：
- 代码通过 Git 管理（最重要）
- 服务器只保存部署脚本与环境变量（不要把 secrets 提交进 Git）

## 3. Docker 场景备份

### 1) PostgreSQL volume

如果你用 Docker volume：
- 备份建议仍使用 `pg_dump`（更通用）

### 2) docker-compose.yml 与 .env

务必备份：
- `docker-compose.yml`
- `.env`（包含数据库密码与连接串，注意权限与保密）
