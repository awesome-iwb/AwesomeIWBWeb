# 内容维护手册

## 维护目标

把内容维护从“改 README 文本”迁移到“结构化数据与后台流程”。

## 日常维护路径

- 单条项目更新：使用后台管理页（`/admin`）
- 开发者改动提案：使用开发者后台（`/dev`）提交审核
- 批量导入：使用 JSON/CSV 导入接口

## 禁止事项

- 禁止继续在根 `README.md` 直接维护项目条目列表。
- 禁止在多个来源重复维护同一字段（例如同时改 runtime 与 seed 且不留说明）。

## 推荐流程

1. 明确本次是“展示修正”还是“数据结构修正”
2. 通过后台或导入脚本更新内容
3. 校验前台列表、详情页与搜索
4. 如需进入构建产物，同步到 `backend/src/data.json`
5. 记录变更来源与影响范围
