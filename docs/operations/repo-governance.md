# 仓库治理约定

## 核心原则

- 本仓库是网页工程仓库，不是 README 文档集合仓库。
- 业务内容必须在结构化数据层维护，不允许回退到根 README 人工堆叠。
- 文档、代码、数据、脚本各自承担明确职责，避免重复真源。

## 变更准入规则

- 功能改动：必须落在 `frontend/` 或 `backend/`，并补充必要文档。
- 内容改动：必须通过后台接口、导入流程或结构化数据文件完成。
- 批量同步：必须记录来源、脚本、执行参数与影响范围。

## PR 最低要求

- 描述改动目的与影响范围
- 提供本地验证结果
- 涉及内容真源调整时，说明 `backend/src/data.json` 与 `backend/runtime/` 的关系处理方式

## 禁止行为

- 将项目条目正文重新写回根 README
- 在多个来源并行编辑同一内容且无说明
- 提交无法复现来源的大规模数据改动
