📊 完整数据模型

users (现有)
├─ id, name, avatar_url, email
├─ role → user | dev | ops ← 保留，dev 由项目归属隐式获得
└─ is_active, created_at, ...

organizations ← 新增
├─ id, name, slug, avatar_url
├─ description, website_url
└─ created_by → users.id, created_at

organization_members ← 新增
├─ org_id → organizations.id
├─ user_id → users.id
├─ role → owner | admin | member
└─ joined_at

project_members ← 新增
├─ project_id → projects.id
├─ member_type → user | organization
├─ user_id → users.id ← 当 member_type=user
├─ org_id → organizations.id ← 当 member_type=organization
└─ role → owner | collaborator

capabilities (现有，扩展)
├─ user:comment ← 新增：发表评论
├─ user:avatar ← 新增：上传/更改头像
├─ user:feedback ← 新增：提交反馈
├─ user:submit_project ← 新增：提交项目
├─ user:profile ← 新增：修改个人资料
├─ dev_panel_access (现有)
├─ admin_panel_access (现有)
└─ ... (现有运维能力不变)

🔄 权限流转逻辑

新用户注册 → 自动获得所有 user:* 能力
提交项目 → 自动成为项目 owner → 获得 dev_panel_access
被邀请加入项目 → 成为 collaborator → 获得 dev_panel_access
加入组织 → 组织下的项目自动获得管理权限
运维分配 → 在项目页指定开发者 → 获得 dev_panel_access
组织者管理 → 组织 owner/admin 可邀请/移除成员

🖥️ 开发者后台功能规划

📋 我的项目
  • 查看自己参与的所有项目
  • 编辑项目信息
  • 管理项目成员（邀请协作者）
🐛 Bug 反馈
  • 查看自己项目的 Bug 列表
  • 修改 Bug 状态（处理中/已修复）
  • 回复 Bug 反馈
💬 评论管理
  • 查看自己项目的评论
  • 删除/屏蔽不当评论
  • 回复评论
📊 项目数据
  • 查看项目访问量
  • 查看评论/反馈统计
  • 趋势图表
🏢 组织管理（组织者可见）
  • 查看/编辑组织信息
  • 邀请/移除组织成员
  • 设置成员角色（admin/member）
  • 管理组织下的项目
