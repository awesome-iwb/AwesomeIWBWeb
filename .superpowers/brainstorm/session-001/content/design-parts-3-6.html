🔐 Part 4：权限流程详解

🟢 用户权限流程
注册 → 自动授予 user:* 能力 → 前端主站正常使用
运维可在后台撤销某个 user:* 能力（如禁止某用户评论）
🔵 开发者权限流程
路径 A：提交项目 → 自动创建 project_members(owner) → 自动提权为 dev → 获得 dev:* 能力
路径 B：被邀请为协作者 → project_members(collaborator) → 自动提权为 dev
路径 C：加入组织 → organization_members → 组织下项目自动获得权限 → 自动提权为 dev
路径 D：申请认领项目 → 运维审核通过 → project_members(owner) → 自动提权为 dev
路径 E：运维在后台指定 → project_members → 自动提权为 dev
🟣 运维权限流程
超级管理员在后台手动分配 → 授予 admin_panel_access + 对应能力
新增：org:review → 审核组织申请
新增：claim:review → 审核项目认领
新增：org:manage → 管理组织状态（暂停/恢复）

🔄 Part 4 续：自动提权/降权逻辑

// 提权：user → dev
async function promoteToDev(userId: string) {
  // 1. 更新 users.role
  UPDATE users SET role = 'dev' WHERE id = userId AND role = 'user';
  // 2. 授予开发者能力
  INSERT dev_panel_access, dev:* capabilities INTO user_capabilities;
  // 3. 发送通知
  CREATE notification (type: role_promoted, title: '已升级为开发者');
}

// 降权：dev → user
async function demoteFromDev(userId: string) {
  // 前提：用户不再是任何项目的成员，也不属于任何组织
  const hasProjects = await checkProjectMembership(userId);
  const hasOrgs = await checkOrgMembership(userId);
  if (hasProjects || hasOrgs) return; // 仍有归属，不降权
  // 检查是否有运维手动授予的 dev 能力
  const manualDevCaps = await getManualDevCapabilities(userId);
  if (manualDevCaps.length > 0) return; // 运维手动给的，不降
  // 执行降权
  UPDATE users SET role = 'user' WHERE id = userId;
  DELETE dev:* capabilities FROM user_capabilities;
  CREATE notification (type: role_demoted, title: '开发者权限已回收');
}

INSERT INTO user_capabilities (user_id, capability_id)
SELECT u.id, c.id FROM users u, capabilities c
WHERE c.id LIKE 'user:%'
ON CONFLICT DO NOTHING;

-- Migration 0025: grant dev caps to existing dev/ops users
INSERT INTO user_capabilities (user_id, capability_id)
SELECT u.id, c.id FROM users u, capabilities c
WHERE u.role IN ('dev', 'ops') AND c.id LIKE 'dev:%'
ON CONFLICT DO NOTHING;

-- Migration 0026: grant new ops caps to existing ops users
INSERT INTO user_capabilities (user_id, capability_id)

WHERE u.role = 'ops' AND c.id IN ('org:review', 'claim:review', 'org:manage')
ON CONFLICT DO NOTHING;
