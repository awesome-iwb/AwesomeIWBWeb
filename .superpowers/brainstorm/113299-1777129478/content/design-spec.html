设计方案：详情页的 Git 风格分支图

下面是具体的技术实现方案。请确认这些细节是否符合你的期望。

1. 数据结构 (Data Structure)

在 data.json 中，为有血缘关系的项目增加 parent 字段：

{
  "name": "Ink Canvas Plus",
  "parent": "Ink Canvas" // 指向被 Fork 的上游项目
}

2. 核心逻辑 (Core Logic)

在前端 ProjectDetailView.vue 渲染时：

  • 遍历 allProjects，找到当前应用的所有上游（Parent）和下游（Children）。
  • 使用 DFS 或简单的树形结构，重建整个“家族树”（Lineage Tree）。

3. 视觉表现 (Visual Design)

我们会使用 HTML/CSS 配合 SVG 连线（或者纯 TailwindCSS Flex 布局）来绘制一个类似 GitHub Network / Git Graph 的节点图：

  • 节点 (Nodes)：显示应用的图标和名称，当前应用高亮显示。
  • 连线 (Edges)：带有方向的曲线，表示派生关系。
  • 交互 (Interactions)：点击任何节点，平滑路由跳转到该项目的详情页。
