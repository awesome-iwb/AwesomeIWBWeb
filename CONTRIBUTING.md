# 贡献指南

感谢你参与 AwesomeIWBWeb。

## 贡献入口

- 功能开发：前端 `frontend/`、后端 `backend/`
- 文档完善：`docs/`
- 内容维护：结构化数据与后台流程（见 `docs/content/maintenance.md`）

## 重要约束

- 本仓库已转型为网页应用工程，不再接受“把业务内容直接堆到根 README”的维护方式。
- 项目条目与分类内容必须通过结构化数据/接口维护。
- 禁止提交非法软件、破解资源或违法内容。

## 开发流程

1. Fork / 新建分支
2. 本地运行前后端并验证
3. 提交变更（代码 + 必要文档）
4. 提交 PR，说明改动目的与验证方法

## 内容类变更建议

- 小规模改动：优先通过后台管理页面处理
- 批量改动：通过导入脚本/同步脚本处理
- 同步外部数据：走飞书快照导入链路，不直接改页面模板

## 审核与安全说明

当前后台鉴权为过渡方案（因 Casdoor 接入条件受限）。  
提交与审核逻辑可用于协作，但不应视为完整生产级权限系统。
