# Awesome Design 后台面板重设计方案

## 一、当前问题分析

### 1.1 侧边栏问题

**AdminSidebar / DevSidebar**（桌面端）：
- ❌ 平铺列表，无分组、无层级，9 个菜单项全部平铺
- ❌ 无折叠/展开功能，菜单项多了就拥挤
- ❌ 无用户信息区域，缺少上下文感知
- ❌ Logo 区域过于简单，缺乏品牌感
- ❌ 退出按钮独立在底部，没有与其他操作整合

### 1.2 底部导航问题

**AdminBottomNav / DevBottomNav**（手机端）：
- ❌ "更多"弹出面板是临时方案，不够优雅
- ❌ 5 个主项 + N 个次项的分配是硬编码的
- ❌ 没有用户头像/信息入口
- ❌ 缺少搜索/快捷操作入口

### 1.3 顶部 Header 问题

**AdminLayout / DevLayout**：
- ❌ Header 过于简单，只有标题 + 返回/退出
- ❌ 手机端 Header 和底部导航功能重叠
- ❌ 没有面包屑导航
- ❌ 没有全局搜索入口

### 1.4 内容区问题

**FloatingPanel**（列表+详情分栏）：
- ❌ 移动端体验差：选中条目后列表消失，需要点"列表"按钮才能切回
- ❌ 列表头部折叠交互不直观（点击展开筛选，但视觉上不明显）
- ❌ 列表项样式不统一（有的用品牌色选中，有的用蓝色）
- ❌ 桌面端左右分栏比例固定，不够灵活

**Dev 后台内容区**：
- ❌ 没有使用 FloatingPanel，而是简单的卡片列表
- ❌ 列表页和详情页之间没有统一的导航模式

### 1.5 整体架构问题

- ❌ Admin 和 Dev 两套后台各自独立实现，大量重复代码
- ❌ 没有统一的"后台框架"组件，每个 Layout 都从零搭建
- ❌ 手机端和桌面端没有独立的、各自优化的体验设计

## 二、设计目标

1. **像 shadcn/ui 一样** — 提供一套统一的后台 UI 组件库，所有后台页面基于同一套组件构建
2. **桌面端** — 侧边栏导航 + 内容区的经典后台布局，支持折叠侧边栏
3. **手机端** — 底部 Tab 导航 + 全屏内容 + 滑动切换，原生 App 级体验
4. **统一框架** — Admin 和 Dev 共享同一套后台框架组件，仅通过品牌色和菜单配置区分

## 三、组件设计

### 3.1 后台框架组件

#### `ui/AdminShell.vue` — 后台外壳（统一框架）

这是整个后台的顶层容器，替代现有的 AdminLayout + DevLayout。

```
┌──────────────────────────────────────────────────┐
│ [桌面]                                            │
│ ┌────────┬───────────────────────────────────┐   │
│ │ Sidebar│  Header                            │   │
│ │        ├───────────────────────────────────┤   │
│ │  Logo  │                                    │   │
│ │  ────  │  Content (router-view)             │   │
│ │  Nav   │                                    │   │
│ │  Items │                                    │   │
│ │  ────  │                                    │   │
│ │  User  │                                    │   │
│ └────────┴───────────────────────────────────┘   │
│                                                    │
│ [手机]                                            │
│ ┌──────────────────────────────────────────────┐ │
│ │ Header (简洁)                                │ │
│ ├──────────────────────────────────────────────┤ │
│ │                                              │ │
│ │  Content (全屏)                              │ │
│ │                                              │ │
│ ├──────────────────────────────────────────────┤ │
│ │ BottomNav                                    │ │
│ └──────────────────────────────────────────────┘ │

```

**Props**：
- `brand`: `'admin' | 'dev'` — 品牌色方案
- `sidebarItems`: 导航菜单配置
- `user`: 当前用户信息
- `title`: 页面标题

#### `ui/Sidebar.vue` — 侧边栏（桌面端）

重新设计的侧边栏：

```
┌──────────────┐
│  ◆ Awesome   │  ← Logo 区域，品牌渐变色
│    后台       │
│  ──────────  │
│              │
│  📊 总览      │  ← 导航项，带图标

│              │
```

- 底部用户信息区（头像 + 名称 + 退出）
- 折叠/展开切换按钮

#### `ui/BottomNav.vue` — 底部导航（手机端）

```
│  总览  项目  审核  用户  更多          │
└──────────────────────────────────────┘
```

**"更多"弹出面板**（从底部滑出的 Sheet）：
```
┌──────────────────────────────────────┐
```

**特性**：
- 主导航 4 个 Tab + "更多"按钮
- "更多"弹出为底部 Sheet（而非浮动卡片），覆盖更多内容
- Sheet 包含：次要导航项 + 用户操作（返回首页/退出）
- Sheet 有拖动条，可下滑关闭
- 选中态使用品牌色

### 3.2 列表框架组件

#### `ui/ListDetailLayout.vue` — 列表+详情分栏布局

替代现有的 FloatingPanel，提供更好的桌面/手机双端体验。

**桌面端**（左右分栏）：
```
┌──────────────┬───────────────────────────┐
│ 🔍 搜索...    │  项目详情                  │
│ ──────────── │  ──────────────────────── │
│ ▸ 项目 A     │  名称: xxx                │
│   项目 B ←选中│  描述: xxx                │
│ ▸ 项目 C     │  ...                      │
│ ▸ 项目 D     │                           │
│              │  [保存] [删除]             │
│ ──────────── │                           │
│  ‹ 1/5 ›    │                           │
└──────────────┴───────────────────────────┘
```

**手机端**（全屏切换）：
```
[列表视图]                    [详情视图]
┌──────────────────────┐    ┌──────────────────────┐
│ 🔍 搜索...            │    │ ← 返回列表            │
│ ──────────────────── │    │ ──────────────────── │
│ ┌──────────────────┐ │    │ 项目详情              │
│ │ 📦 项目 A        │ │    │ 名称: xxx             │
│ └──────────────────┘ │    │ 描述: xxx             │
│ ┌──────────────────┐ │    │ ...                   │
│ │ 📦 项目 B  ←选中 │ │    │                      │
│ └──────────────────┘ │    │ [保存] [删除]         │
│ ┌──────────────────┐ │    └──────────────────────┘
│ │ 📦 项目 C        │ │
│ └──────────────────┘ │
│ ──────────────────── │
│  ‹ 1/5 ›            │
└──────────────────────┘
```

**Props**：
- `items`: 列表数据
- `selectedId`: 当前选中项 ID
- `searchable`: 是否显示搜索框
- `searchPlaceholder`: 搜索框占位文字
- `page`, `total`, `pageSize`: 分页

**Slots**：
- `list-item`: 列表项渲染
- `detail`: 详情内容
- `empty-detail`: 未选中时的占位内容

**手机端交互**：
- 点击列表项 → 滑入详情视图（从右滑入）
- 点击"返回" → 滑回列表视图
- 使用 CSS transition 实现滑动动画

### 3.3 列表项组件

#### `ui/ListItem.vue` — 统一列表项

```
┌──────────────────────────────────┐
│ 📦  项目名称                      │
│     描述文字...                   │
│                          [状态]   │
└──────────────────────────────────┘
```

**Props**：
- `title`: 标题
- `subtitle`: 副标题/描述
- `icon`: 左侧图标
- `active`: 是否选中
- `badge`: 右侧状态徽章文字
- `badgeStatus`: 状态类型

**选中态样式**：
```
bg-[var(--color-brand-500)] text-white shadow-md shadow-[var(--color-brand-shadow)]
```

**未选中态样式**：
```
bg-white dark:bg-slate-800 hover:bg-slate-50 dark:hover:bg-slate-700/50
border border-slate-200 dark:border-slate-700
```

## 五、实施步骤

### Step 1：创建 BottomSheet 组件
- 底部弹出面板，支持拖动关闭
- 用于"更多"菜单和移动端的各种弹出操作
- 毛玻璃背景 + 圆角 + 阴影

### Step 2：创建 ListItem 组件
- 统一列表项样式
- 支持选中态（品牌色背景 + 阴影）
- 支持图标、标题、副标题、状态徽章

### Step 3：创建 Sidebar 组件
- 支持导航分组
- 支持折叠/展开（图标模式）
- 底部用户信息区
- 通过 CSS 变量自动适配品牌色

### Step 4：创建 BottomNav 组件
- 主导航 Tab + "更多"按钮
- "更多"弹出 BottomSheet
- Sheet 包含次要导航 + 用户操作

### Step 5：创建 BackHeader 组件
- 桌面端：标题 + 用户名 + 操作按钮
- 手机端：返回箭头 + 标题

### Step 6：创建 ListDetailLayout 组件
- 桌面端：左右分栏
- 手机端：全屏切换 + 滑动动画
- 内置搜索框 + 分页器

### Step 7：创建 AdminShell 组件
- 整合 Sidebar + BottomNav + BackHeader
- 通过 brand prop 区分 admin/dev
- 通过 sidebarItems prop 配置菜单

### Step 8：重构 AdminLayout + DevLayout
- AdminLayout 使用 `<AdminShell brand="admin" :sidebar-items="adminItems" />`
- DevLayout 使用 `<AdminShell brand="dev" :sidebar-items="devItems" />`
- 删除旧的 AdminSidebar、AdminBottomNav、DevSidebar、DevBottomNav

### Step 9：重构内容页面
- 将使用 FloatingPanel 的页面迁移到 ListDetailLayout
- 将使用内联列表的页面迁移到 ListItem
- 统一所有列表项的选中态样式

### Step 10：验证 + 清理
- 类型检查 + 构建验证
- 删除不再使用的旧组件
- 手机端和桌面端分别测试

## 六、关键设计规范

### 侧边栏

| 属性 | 值 |
|------|-----|
| 宽度（展开） | `w-56` (224px) |
| 宽度（折叠） | `w-16` (64px) |
| 背景 | `bg-white dark:bg-slate-800` |
| 边框 | `border-r border-slate-200 dark:border-slate-700` |
| 导航项圆角 | `rounded-xl` |
| 导航项内边距 | `px-3 py-2.5` |
| 选中态 | `bg-[var(--color-brand-500)] text-white shadow-md shadow-[var(--color-brand-shadow)]` |
| 悬浮态 | `hover:bg-slate-100 dark:hover:bg-slate-700` |
| Logo 区内边距 | `p-4` |
| 用户区内边距 | `p-3` |

### 底部导航

| 属性 | 值 |
|------|-----|
| 高度 | `h-16` (64px) |
| 背景 | `bg-white dark:bg-slate-800 border-t` |
| Tab 图标 | `w-5 h-5` |
| Tab 文字 | `text-[10px] font-medium` |
| 选中态 | `text-[var(--color-brand-500)]` |
| 未选中态 | `text-slate-400` |

### BottomSheet

| 属性 | 值 |
|------|-----|
| 最大高度 | `max-h-[70vh]` |
| 圆角 | `rounded-t-3xl` |
| 背景 | `bg-white dark:bg-slate-800` |
| 阴影 | `shadow-2xl` |
| 拖动条 | `w-10 h-1 rounded-full bg-slate-300 dark:bg-slate-600` |

### ListDetailLayout

| 属性 | 值 |
|------|-----|
| 列表面板宽度 | `w-80` (320px) |
| 列表面板背景 | `bg-slate-50 dark:bg-slate-900/50` |
| 详情面板背景 | `bg-white dark:bg-slate-800` |
| 分隔线 | `border-r border-slate-200 dark:border-slate-700` |
| 手机端动画 | `transition-transform duration-300 ease-out` |
