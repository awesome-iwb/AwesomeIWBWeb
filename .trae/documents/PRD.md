# Awesome-IWB 网页版 产品需求文档 (PRD)

## 1. 简介
### 1.1 背景
Awesome-IWB 目前是一个托管在 GitHub 上的开源软件推荐清单（Markdown 格式），主要服务于中小学电教老师，提供班级一体机（如希沃、鸿合等）的优质软件工具。随着收录项目增多（目前 README 超过 4500 行），单文件维护成本极高，用户浏览体验受限，且难以实现高级搜索、筛选和动态展示。因此，急需开发一个结构化、交互友好的网页版本。

### 1.2 目标
构建一个现代化、响应式、交互友好的静态站点，用于展示 Awesome-IWB 的软件列表。
- 提供比 GitHub Markdown 更出色的视觉体验和交互（如卡片式展示、动画效果）。
- 支持基于数据驱动（YAML/JSON）的自动化内容渲染，便于长期维护。
- 提供分类浏览、标签筛选、关键字搜索等高级功能，帮助教师快速找到所需软件。

## 2. 目标受众
- **中小学电教老师/系统管理员**：寻找能够优化班级多媒体教学体验的软件。
- **开源开发者**：提交自己的软件项目到推荐列表中。
- **学生**：寻找一些能够改善班级屏幕体验的辅助工具（如倒计时、课表）。

## 3. 核心功能需求
### 3.1 首页展示
- **Hero Section**：醒目的项目 Logo（支持暗黑/明亮模式切换）、Slogan（"专为广大中小学电教打造的一站式软件推荐清单"），以及核心行动号召按钮（如“浏览项目”、“贡献指南”）。
- **数据统计/状态栏**：展示收录项目总数、GitHub Star 数、最后更新时间等。
- **项目分类快速入口**：以卡片形式展示核心分类（屏幕批注与白板、课表与看板、辅助类软件等）。

### 3.2 软件列表展示区
- **分类导航**：左侧/顶部粘性导航栏，支持在不同分类间平滑切换。
- **项目卡片展示**：每个软件呈现为一个独立卡片，包含：
  - 软件名称与推荐等级（🥇/🥈/🌟/🔴）
  - 开发者头像与名称
  - 项目简介与功能特点
  - 关键词/标签（Badge）
  - GitHub 活跃度状态（动态拉取或预编译渲染）
  - 用户评价（“锐评”）
  - 下载/访问链接图标
- **状态过滤与搜索**：
  - 文本搜索框：通过名称或描述搜索软件。
  - 标签筛选：按“推荐等级”、“软件类型（如屏幕批注）”、“活跃状态”进行筛选。

### 3.3 贡献与关于页面
- 整合 `CONTRIBUTING.md` 内容，提供清晰的提交规范说明。
- 提供社区交流群（QQ群）的二维码/入口。

## 4. 设计与审美方向
- **风格**：现代化、干净、富有科技感但保持亲和力（面向教育行业）。建议采用带有细微毛玻璃效果（Glassmorphism）、大圆角卡片、平滑过渡动画的现代极简主义设计。
- **排版**：使用易读的现代无衬线字体（如 Inter 或类似字体），标题醒目，内容层次分明。
- **色彩**：支持 Light / Dark 双主题。主色调可参考原项目的蓝色/教育科技感色调，配合醒目的强调色（如橙色代表“非常推荐”）。

## 5. 验收标准
- 页面加载迅速，核心内容首屏可见。
- 响应式设计完美适配桌面、平板和移动端（教师可能使用手机查阅）。
- 搜索与筛选功能响应延迟极低（客户端实现）。
- 视觉效果大幅超越原生 GitHub Markdown 页面。
